# type

## 作用

- 把参数值转换为目标类型，避免 SQL/模板/模型写入时类型不匹配。
- 常见：字符串 `"1"` 转整数 `1`，字符串 `"true"` 转布尔 `true`。

## 常见用途

- 把参数值转换为目标类型，避免 SQL/模板/模型写入时类型不匹配。
- 常见：字符串 `"1"` 转整数 `1`，字符串 `"true"` 转布尔 `true`。

## 执行阶段（低代码视角）

- 参数/处理器执行阶段：在参数计算、条件判断或处理器运行时生效。

## 怎么用

### 配置位置
- 主要用于 `params[字段].type`
- 也可出现在部分处理器字段规则中（如 `update_array.fields[].type`）

### 常见类型
- `bool`
- `int` / `int64`
- `float` / `float64`
- `string`
- `json`（部分引擎支持）

## 示例

```yml
params:
  page:
    type: int
    default: 1
  pagination:
    type: bool
    default: true
```

## 注意事项

- 建议与 `default` 搭配，保证空值场景也能稳定转换。
- 类型转换失败时，不同引擎处理方式不同（忽略/报错），建议通过自测确认。
