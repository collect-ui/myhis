### 老系统关键联动顺序

1. 打开弹框：`open()` 调 `pixGetFee()` 和 `setDefault()`。
2. `setDefault()`：重置表单；从上下文带出 `areaCode/areaName`；加载科室、专科；默认排班日期为今天；指定新增时走 `addSimple` 分支。
3. 选择科室 `orgChange()`：设置急诊标志，清空专科/医生/平均时间，`registrationType` 回到 `0`，重新加载专科和医生。
4. 选择专科 `clinicChange()`：清空平均时间，`registrationType` 回到 `0`，用专科对象反填 `resideDeptCode`，重新加载医生。
5. 选择医生 `doctorChange()`：重新获取费别；如果当前费别不在新费别列表则清空费别和号别；未选专科时通过医生反查专科；最后触发查询医生排班。
6. 选择费别 `selectRegFee()`：带出号别；急诊时强制按时段；若当前左侧已选时段且为平均看诊模式，则重算右侧分时号源。
7. 改限号设置 `registrationTypeChange()`：只有切到 `1` 时才需要 `averageTime`；没有医生时从系统参数 `dept_master_average_time_outp_master` 按专科取默认值，有医生时取医生-专科维护平均时间。
8. 选择排班日期或点击“查询医生排班”：`findDoctorSchedule({ areaCode, doctorCode, outpDate, resideDeptCode, specialClinicCode })`；没有既有排班时插入一行新时段。
9. 开启分时标志：按时段模式调 `/pix/regMasterAtime/getDuration/{outpDurationCode}`，按平均看诊模式调 `/pix/pixOutpRegSchedule/generateAtimeByAvgVisitTime`。
10. 提交：`pixAddTemp()` -> `/pix/pixOutpRegMaster/batchAdd`，提交结构是 `form + batchRegMasterDurationDTOList`，列表元素里包含每个时段及其 `regMasterAtimeVOList`。

### 时间字段关系

| 字段 | 老系统 | 当前低代码 | 目标 |
| --- | --- | --- | --- |
| 排班日期 | `form.outpDate`，格式 `yyyy-MM-dd` | `opForm.outp_date`，`valueFormat: YYYY-MM-DD` | 保持 `YYYY-MM-DD`，新增时默认今天；禁选今天以前 |

### 已确认的当前问题定位

1. 院区：当前 SQL 只返回编码，导致页面显示 `H001`；老系统显示上下文里的中文 `areaName`。当前只有一条时也没有默认选中。
2. 挂号费别：当前直接查 `pix_reg_type_fee`，按名称排序且包含测试/异常名称，如 `aaa`；老系统通过业务接口 `/pix/pixRegTypeFee/findAllByParams`，传 `areaCode/isEnable/doctorCode`，返回口径和排序可能不同，“普通号”缺失应优先查这里。
3. 平均看诊时间：当前普通字段一直可见；老系统只有选择“按平均看诊时间生成”才显示并参与计算。
4. 星期选择：当前靠前端表达式即时计算，已经出现过表达式文本外露；应改成选择日期时写入 store 字段。
5. 分时详情：老系统有右侧分时明细表；当前主时段仍只录入一条主表记录，但选择 `分时标志=启用` 后恢复 `分时详情` input-table，并按老系统默认时间点生成分时段。
6. 新增临时排班提交：当前 `model_save pix_outp_reg_master` 只能保存单条主表；老系统提交 `batchRegMasterDurationDTOList`。如果要完全等价，需要新增/编排批量保存主表 + 分时明细的低代码服务，或确认是否已有分时明细表服务。

1. 先把“新增临时排班”弹框和“编辑号源”拆清楚：新增临时排班只保留老弹框字段，编辑号源再保留 `is_enable/master_status/sort_no/alias_name/descn`。

   - 修改 `collect/him/pix_outp_reg_master/area_options.sql`，院区来源改为 `bcs.bcs_hospital_area`。
   - 返回结构补齐 `{ label: area_name, value: area_code, area_code, area_name }`。
   - 当前唯一院区 `H001` 在新增弹框默认选中并显示 `主院区`。

2. 挂号费别排序与普通号可见性
   - 修改 `collect/him/pix_outp_reg_master/reg_fee_options.sql`，下拉 label 改为业务名称，不再显示 ID 干扰选择。
   - 排序改为“普通号”优先、测试/aaa 类靠后。
   - 数据库核对结果：`普通号`、`中医普通号`、`药学普通号` 当前 `delete_flag=1`，可用数据里只有 `普通号（夜间）`，因此未伪造不可用选项。

3. 平均看诊时长显示条件
   - 当前低代码页 `collect/frontend/page_data/data/his/pix_outp_reg_master.json` 中 `average_time` 增加 `itemVisible`。
   - 初始按时段模式不再显示“平均时长”，只有选择“按平均看诊时间生成”才显示。

4. 星期选择表达式外露
   - 增加稳定 store 字段 `opFormWeekText`。
   - 日期组件变更时写入星期文本。
   - 星期展示改为只读文本容器，不再把 `${opFormWeekText}` 塞进 input 的 `value`，避免表达式或空值外露。

5. 临时号源单条录入与分时详情
   - 按当前确认，新增临时号源主时段只录入一条主表记录，不再使用 `input-table`。
   - 新增模式直接显示单条字段：
     - `outp_duration_code`
     - `registration_limits`
     - `appointment_limits`
     - `transfer_no`
     - `internet_limits`
     - `atime_flag`
   - 新增提交校验改为直接校验 `outp_duration_code` 和 `registration_limits`。
   - 新增保存不再从 `duration_rows[0]` 回填，直接提交单条表单字段。
   - `分时标志` 选择 `启用` 后显示 `分时详情` input-table，用于录入分时时段、开始/结束时间、限号数、预约数、预留数、互联网。
   - 分时段按老系统默认 `atime_master_timepoint = 2-9;10-12;13-19;20-33;34-1;34-33` 生成，班次顺序为 `上午/中午/下午/晚上/凌晨/全天`。
   - 数量分摊按老系统 `calcLimts` 规则：每个分时段先平均分配，余数从前面的分时段依次 +1；例如下午班次限号/预约均为 14 时，首段 `13:30-14:00` 自动填 `2 / 2`。

6. 日期选择与日期展示
   - 排班日期组件显式配置 `picker: date`，保持年月日日期选择。
   - 调整字段顺序，默认按时段模式下同一行显示 `限号设置 / 排班日期 / 星期选择`，避免用户只看到星期字段。
   - 日期输入仍使用后端需要的 `YYYY-MM-DD` 值，不提交带时间的字符串。
   - 列表 `门诊日期` 增加 `valueFormatter`，只显示 `YYYY-MM-DD`，不再显示 `00:00:00 +0000 UTC`。

### 保留边界

- 当前低代码后端仍只有 `him.pix_outp_reg_master_save` 的单条 `model_save` 能力。
- 当前页面按单条临时号源主表录入，不采集 `duration_rows`。
- 当前页面会在 `分时标志=启用` 时按老系统默认时间点生成并采集 `atime_rows`，但现有单条 `model_save` 暂不落 `pix_outp_reg_master_atime` 明细表。
- 如果后续要完全等价老系统 `/pix/pixOutpRegMaster/batchAdd` 的多时段与分时明细，还需要另行确认需求并补低代码批量保存主表与 `pix_outp_reg_master_atime` 明细的服务编排。
