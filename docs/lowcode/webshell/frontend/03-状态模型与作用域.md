# 状态模型与作用域

## 1. 作用域原则

- 顶层 store：保存页面级状态（如 drawer 开关、项目列表、当前项目）
- pane 局部 store：保存工作区运行态（tree/fileMap/panelList/activePath 等）

## 2. 为什么要局部 store

- 避免多个项目 pane 共享同一套运行态，产生“串状态”
- 保证分割、打开文件、滚动位置在项目内独立

## 3. 必须放顶层的字段

- `workspaceOptions`
- `workspaceCurrent`
- `workspaceLoadedMap`
- `workspaceVisible`
- `workspaceDrawerMode`
- `workspaceDrawerWidth`

## 4. 必须放 pane 局部的字段

- `workspaceTree`
- `workspaceFileMap`
- `workspacePanelList`
- `workspaceTreeSelected`
- `workspaceLastGroupId/workspaceLastShellId`
- `workspaceActivePath/workspaceActiveName/workspaceToken`

## 5. 上下文变量约定

- 项目标识优先使用 `row.path`
- 无 `path` 时回退 `row.value`
- 不要在同一页面混用 `row.path` 与 `row.value` 作为主键
