# Frontend 快速入门

## 1. 这个 webshell 是做什么的

`webshell` 是业务侧的“服务器运维 + 项目工作空间”页面，前端低代码配置承担三类能力：

1. 服务器连接与终端标签管理（SSH）
2. 文件浏览与操作历史回放
3. 工作空间（workspace）项目切换、目录树浏览、文件编辑与分割布局

## 2. 页面布局结构（高频改动区）

- 左侧 `sider`：服务器树、登录入口、节点增删改
- 右侧 `content`：主 SSH 区（panelList + tabs）
- 右上角浮动按钮：打开工作空间 drawer
- `drawer` 内：项目切换 + workspace tabs + tree + editor + 分割能力

## 3. 核心入口文件

- `collect/frontend/page_data/data/server/webshell.json`

## 4. 优先理解的状态字段

- `workspaceOptions`: 项目列表（支持动态新增）
- `workspaceCurrent`: 当前项目 key
- `workspaceLoadedMap`: 已加载项目映射（懒加载 tabs）
- `workspaceVisible/workspaceDrawerWidth/workspaceDrawerMode`: drawer 状态与宽度

> 注意：workspace 的运行态（tree/fileMap/panelList 等）在 tabs 的 `layout-fit` 局部 store 中，不在顶层全局 store。

## 5. 优先理解的动作链

- 项目切换：`rowClickAction` -> 更新 `workspaceCurrent + workspaceLoadedMap`
- tree 点击文件：`selectAction` -> 写入 activePath/token -> 打开或激活文件 tab
- editor tabs 关闭：`removeAction`
- 分割：`rightMenuAction`（水平 arr-push，垂直 addChildren）

## 6. 新需求改动顺序（建议）

1. 先改状态字段（store）
2. 再改动作链（action）
3. 最后改渲染层（itemData/children）

## 7. 5 分钟回归检查

1. 切项目是否生效（activeKey + loadedMap）
2. 点树文件是否打开 tab
3. 水平/垂直分割是否都生效
4. 关闭 tab 后其他面板是否仍显示内容
5. 刷新后 drawer 宽度与显示状态是否保持
