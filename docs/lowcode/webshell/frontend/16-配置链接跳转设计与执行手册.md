# Webshell Editor Pool 配置链接跳转设计与执行手册

## 1. 目标

在 `webshell-editor-pool` 中实现类似 IDEA 的“配置可跳转”能力，针对你当前后端低代码目录（重点是 `/data/project/moongod-backend/backend_data_service`）支持点击后快速定位：

1. `service_router.yml` 里的 `path` 跳转到目标 `service.yml`。
2. `service.yml / index.yml` 里的 `sql_file / count_sql` 跳转到目标 SQL。
3. `service2field` 中的 `service: module.service_key` 跳转到对应服务定义 `- key: service_key` 所在行。
4. `data_json` 跳转到同目录 JSON 文件。
5. SQL 中 `require("./xxx.sql")` 跳转到被引入 SQL 文件。
6. `model: XxxModel` 跳转到 `backend/models/models.py` 中对应 class 行。

## 2. 交互定义

在编辑器内对可识别字段触发跳转：

1. `path: 'project/service.yml'`
2. `sql_file: issue_commit_detail.sql`
3. `count_sql: count.sql`
4. `service: jira.issue_commit_count`
5. `data_json: query_black_list.json`
6. `require("./base.sql")`
7. `model: MsgSendRule`

触发方式（IDEA 风格）：

1. 按住 `Ctrl`（macOS 为 `Command`）悬停在可跳转值上，出现下划线提示。
2. 保持 `Ctrl`（或 `Command`）并左键点击，执行跳转。

触发后行为：

1. 自动打开目标文件（若已打开则切换到目标 tab）。
2. 若有目标行号（`service` 场景），自动定位并高亮该行。
3. 无法解析时给出明确错误提示，不破坏当前编辑状态。

## 3. 规则与解析策略

### 3.1 文件路径类跳转

1. `path` 字段：当值是相对文件路径（例如 `xxx/service.yml`、`xxx/index.yml`）时，基于当前文件目录拼接目标路径。
2. `sql_file / count_sql` 字段：基于当前文件目录拼接目标 SQL 路径。
3. `data_json` 字段：基于当前文件目录拼接目标 JSON 路径。
4. `require("./xxx.sql")`：基于当前 SQL 文件目录拼接目标 SQL 路径。

### 3.2 服务引用类跳转

输入：`service: module.service_key`

解析流程：

1. 定位当前项目根下 `backend_data_service/service_router.yml`。
2. 从 `service_router.yml` 找到 `module -> module/service.yml` 映射。
3. 从该 `module/service.yml` 读取子 `index.yml` 列表（`path` 字段）。
4. 扫描每个 `index.yml`，定位 `- key: service_key` 行号。
5. 打开对应 `index.yml` 并跳转到命中行。

### 3.3 Django Model 跳转

输入：`model: MsgSendRule`

解析流程：

1. 基于当前文件定位 `backend_data_service/service_router.yml`。
2. 读取 `django_model.model_file`（当前为 `backend.models.models`）。
3. 将模块名转换为 Python 路径（如 `backend/models/models.py`）。
4. 在目标文件中查找 `class MsgSendRule(...)` 行并跳转。

### 3.4 缓存策略

为避免频繁网络请求，前端内存缓存以下数据（按 `project_code + path` 维度）：

1. 文件文本缓存。
2. `service_router.yml` 模块映射缓存。
3. `module/service.yml` 到 `index.yml` 列表缓存。
4. `module.service_key` 到 `{ path, line }` 的解析缓存。

## 7. 风险与边界

1. 若配置值不是文件路径（例如普通 URL path），不会生成跳转。
2. 若 `service_router.yml` 中模块未配置，`service` 跳转会提示解析失败。
3. 若同名 `service_key` 在同模块多个文件重复，默认命中首个匹配项。

## 10. 项目类型与 Python 包路径配置（2026-05-07）

为避免 Python 源码路径写死，项目管理新增两项配置字段（落库到 `webshell_workspace_project`）：

1. `project_type`：项目类型（`go` / `python`）。
2. `python_pkg_path`：Python 安装包根路径（例如 `/data/soft/python27`，或直接填 `.../site-packages`）。

设计原则：

1. 跳转逻辑不再内置固定 Python 路径。
2. 优先使用项目配置字段。
3. 若未配置，则尝试解析项目启动脚本中的 `PY_HOME`（如 `linux-start-dev` / `linux-startup` / `startup.sh`），自动推导 `site-packages`。
