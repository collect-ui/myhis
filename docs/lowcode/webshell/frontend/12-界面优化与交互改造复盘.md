# Webshell 界面优化与交互改造复盘

本文总结本次 `webshell.json` 的前端低代码改造思路、技术点与实践经验，便于后续迭代复用。

## 1. 改造目标

- 在不改后端接口的前提下提升界面观感（层次感、统一色阶、卡片化）。
- 提升高频操作可达性（双击不稳定时提供右键登录兜底）。
- 保持布局紧凑（尤其 Drawer 内容区不留默认 padding）。
- 兼顾大数据列表可读性（默认展开层级更保守、首页改卡片聚合展示）。

## 2. 主要改造点

### 2.1 服务器树与右键能力增强

- 主服务器树默认展开层级由 `openLevel: 2` 调整为 `openLevel: 1`，减少初始信息噪音。
- 在三个入口补齐右键登录：
  - 左侧服务器列表
  - 常用服务器
  - 快速开始区域
- 右键登录动作链与双击登录一致：
  - 打开登录弹窗
  - 重置登录表单
  - 回填服务器展示字段
  - 拉取 OS 用户
  - 自动选中默认用户

### 2.2 快速开始区域卡片化

- 将“快速开始”中的服务器从树形改为卡片列表，提升扫读效率。
- 卡片支持多列（目标 2~3 列，按宽度自适应降级）。
- 单卡包含：服务器图标、IP/业务名、环境与使用次数、登录按钮。
- 无数据时显示明确空态提示。

### 2.3 TOP 标识（常用服务器）

- 对常用前 5 个服务器打 `TOP1~TOP5` 标识。
- 样式采用左侧附着式“红旗彩带”，强调常用优先级。
- 最终位置策略：
  - 视觉附着在卡片左侧边框
  - 不占内容流空间（绝对定位）
  - 通过小尺寸字重与轻阴影保持高级感但不过度抢眼

### 2.4 紧凑布局治理

- 明确区分两类留白：
  - 需要保留：工作空间标题工具组（项目切换、宽度切换）可操作间距
  - 需要移除：`ant-drawer-body` 默认 padding
- 实施方式：为 Drawer 增加 `bodyStyle: { "padding": 0 }`。

## 3. 视觉设计原则（本次落地）

- 色阶统一：浅蓝灰中性色为主，局部高亮用于引导（登录按钮、TOP 彩带）。
- 层次建立：通过轻边框 + 渐变底色 + 轻阴影构建卡片层级。
- 信息优先：主信息（服务器）> 次信息（环境/次数）> 操作按钮。
- 避免过度装饰：标识可见但不喧宾夺主，控制字号、padding 与阴影强度。

## 4. 技术经验与踩坑

### 4.1 低代码表达式复用注意事项

- 同一登录动作分散在多个入口时极易漂移，建议抽 action group 统一维护。
- 但实际替换时要验证运行时上下文变量（如 `node`）是否一致，避免入口可用性回退。

### 4.2 布局引擎差异

- 某些组件（如 listview）对子元素布局能力有限，`flex` 多列可能不生效。
- 实践中通过 `inline-flex + width + minWidth + margin` 更稳定实现多列卡片。

### 4.3 悬浮标识定位

- “看起来附着”不等于“视觉正确”：
  - 需要同时控制 `top/left`、`zIndex`、`transform`。
  - 若出现内容被挤压，优先检查是否误加内边距补位。
- 最终采用绝对定位并移除内容补位，实现“装饰不占位”。

### 4.4 保持可用性优先

- 任何视觉改造都要守住操作链路：
  - 双击登录可用
  - 右键登录可用
  - 登录后用户自动带出可用

## 5. 推荐后续优化

- 将“登录动作链”沉淀为统一 action group，减少维护成本。
- 为快速开始卡片增加 hover 态（轻阴影/描边）与键盘可访问性。
- 新增“提示可关闭并记忆”能力，降低老用户视觉干扰。
- 增加 `TOP` 规则可配置（按最近登录次数/最近时间切换）。

## 6. 回归检查清单

- 双击登录：主树、常用树、快速开始卡片都可用。
- 右键登录：三个入口均可触发完整动作链。
- 快速开始卡片：宽屏 2~3 列，窄屏自动降级。
- Drawer：body 无额外 padding，工具区间距仍可用。
- JSON 语法校验通过。
