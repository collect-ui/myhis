# 01 运行时菜单与页面链路

当前项目的真实链路不是静态 `menu.json`。

运行时关键来源：

- 菜单：`sys_menu`
- 权限：`role_menu`
- 页面 service 映射：`collect/frontend/page_data/index.yml`
- 页面 JSON：`collect/frontend/page_data/data/**/*.json`
- 菜单查询接口：`system.menu_query`

补充规则：

- 若菜单设置 `is_common=1`，则 `system.menu_query` 会直接返回该菜单
- 这类公共菜单不需要额外写 `role_menu`
- 只有非公共菜单才依赖 `role_menu`

判断页面为何打不开时，优先检查：

1. `sys_menu.api` 是否指向正确的 `frontend.*`
2. `page_data/index.yml` 是否存在对应 `key`
3. JSON 文件路径是否正确
4. 若 `is_common!=1`，再检查当前角色是否在 `role_menu` 里拥有该菜单
