# 04 邮箱登记页面实战

本实战对应文件：

- 模型：`model/base/mail_account.go`
- 后端：`collect/system/mail_account/`
- 页面：`collect/frontend/page_data/data/system/mail_account.json`
- 菜单 seed：`test/lowcode-page/scripts/sql/mail_account_seed.sql`

业务链路：

1. 前端读取多行原始文本
2. 前端按 `----` 解析出 `valid_list`
3. 前端同步生成 `error_list`、`duplicate_in_batch_list`、`email_name_list_sql`
4. 后端通过 `system.mail_account_import_batch` 查询库内已存在邮箱
5. 后端把存在项拆到 `skip_list`
6. 后端把新增项交给 `system.mail_account_bulk_create`
7. 页面刷新列表并显示导入摘要

关键约束：

- 不新增专用 Go 导入模块
- 同批重复在前端拦截
- 库内重复在后端跳过
- 删除按物理删除实现
- 列表默认明文展示密码和恢复码
