# 02 需求分析与组件选型流程

开始写页面前，先把需求拆成“页面行为”和“后端行为”两部分。

页面行为分析：

- 是否只是查询列表
- 是否需要前端预处理或解析
- 是否需要分页、删除、导入结果提示

后端行为分析：

- 是否可以直接复用 `model_save` / `model_update` / `model_delete`
- 是否适合走 `empty + service2field + update_array + filter_arr + bulk_create`
- 是否真的需要新增 Go 模块

组件选型优先查：

- `/data/project/collect-ui/docs/readme/components/layout-fit.md`
- `/data/project/collect-ui/docs/readme/components/form.md`
- `/data/project/collect-ui/docs/readme/components/input.md`
- `/data/project/collect-ui/docs/readme/components/table.md`
- `/data/project/collect-ui/docs/readme/components/confirm.md`

邮箱登记页的结论：

- 页面容器：`layout-fit`
- 查询：`form + input`
- 批量文本：`input` with `isTextarea`
- 结果提示：`alert`
- 列表：`table + pagination`
